## `$ cat overview.txt`

Codexa is a gamified DSA practice platform with a **WebSocket-heavy backend** and an **automated cron job system**. The real-time layer powers live help sessions, room-based chat, and presence tracking. The progression engine (XP, streaks) runs entirely server-side. Badge and leaderboard models exist in the schema but are not yet implemented. Cron jobs auto-assign daily targets and generate daily challenges at midnight IST.

### Data Models

```
User
  в”ңв”Җв”Җ name, username (unique), email (unique)
  в”ңв”Җв”Җ passwordHash, xp, streak, problemsSolved
  в”ңв”Җв”Җ helpCount, dailyTarget, lastStreakDate
  в”ңв”Җв”Җ levelId вҶ’ Level
  в””в”Җв”Җ badges  вҶ’ [Badge]

DSAProblem
  в”ңв”Җв”Җ title (unique), link, topic
  в”ңв”Җв”Җ difficulty, xpReward
  в””в”Җв”Җ sheetName, orderInSheet

UserProblemLog              вҶҗ solve status per user per problem
  в”ңв”Җв”Җ user вҶ’ User
  в”ңв”Җв”Җ problemId (polymorphic) вҶ’ DSAProblem | RandomProblem
  в”ңв”Җв”Җ status: -1 | 0 | 1 | 2
  в””в”Җв”Җ xpAwarded, solvedAt

InstantSolveSession         вҶҗ real-time help session state
  в”ңв”Җв”Җ userId вҶ’ User
  в”ңв”Җв”Җ status: available | doubt | in-session | offline
  в””в”Җв”Җ socketId, roomId, lastMatchedUser

HelpSessionLog              вҶҗ completed session analytics
  в”ңв”Җв”Җ helper вҶ’ User, asker вҶ’ User
  в”ңв”Җв”Җ roomId, duration (minutes)
  в””в”Җв”Җ status, startedAt, endedAt, endedBy
```

### Problem Status System
```
-1  вҶ’  backlog  (red)
 0  вҶ’  today    (blue)  вҶҗ auto-assigned by cron at midnight
 1  вҶ’  solved   (green)
 2  вҶ’  future   (grey)
```

```
Help Request Flow:
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
User A (asker)          SocketManager           User B (helper)
     в”Ӯ                       в”Ӯ                        в”Ӯ
     в”ңв”Җв”Җ askHelp в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө                        в”Ӯ
     в”Ӯ                       в”ңв”Җв”Җ find available helperв”Ӯ
     в”Ӯ                       в”ңв”Җв”Җ create roomId        в”Ӯ
     в”Ӯ                       в”ңв”Җв”Җ update sessions      в”Ӯ
     в”Ӯ                       в”ңв”Җв”Җ emit "matched" в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө
     в”ңв”Җв”Җ matched в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө                        в”Ӯ
     в”ңв”Җв”Җ join-room в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө                        в”ңв”Җв”Җ join-room
     в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ Chat session active в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

Room ID format: userId_helperId_timestamp

Socket Events:
  join-room         вҶ’  validate + enter private room
  send-message      вҶ’  broadcast text/file to room
  receive-message   вҶ’  incoming message handler
  typing            вҶ’  typing indicator broadcast
  end-session       вҶ’  terminate + update HelpSessionLog
  disconnect        вҶ’  cleanup InstantSolveSession
```

### Cron Job System

```
Midnight IST вҖ” dailyTarget.cron.js
  в”ңв”Җв”Җ query all users
  в”ңв”Җв”Җ select unsolved problems per user
  в”ңв”Җв”Җ set status: 0 (today's target)
  в””в”Җв”Җ create UserProblemLog entries

Midnight IST вҖ” dailyChallenge.cron.js
  в”ңв”Җв”Җ select 5 random problems
  в””в”Җв”Җ create DailyChallenge { date, randomProblemIds }
```
